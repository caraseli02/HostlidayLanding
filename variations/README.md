# Landing Page Concepts — Three Genuinely Different Strategies

Each concept uses a **different section skeleton**, **different hero framing**, **different proof model**, and **different conversion logic**. They are not visual skins — they are strategic directions for review before publishing.

---

## Concept 1: Conversational Trust Assistant (`intercom/`)

**Inspiration:** Intercom

**Hero framing:** "Meet your planner" — personal, warm, the page *is* a conversation preview

**Section order:** Hero → Chat demo → Before/after comparison → Trust promises → CTA

**Proof structure:** Live chat conversation demonstrating the product working in real-time. The product IS the proof — you see a verified recommendation arrive in a chat bubble.

**Conversion logic:** Emotional pull. You experience the product on the page itself. The CTA extends the conversation metaphor ("start planning with confidence").

**Emotional tone:** Warm, personal, approachable. "We're in this together."

---

## Concept 2: Verification Infrastructure (`rippling/`)

**Inspiration:** Rippling

**Hero framing:** Split layout — claim on left, live verification dashboard on right. System-first.

**Section order:** Hero (with data card) → Stats bar → Verification pipeline (4-step) → Problems/solutions table → CTA

**Proof structure:** Quantitative metrics (100% source-backed, <48h freshness, 5 checks per pick, 0 unsourced) + a 4-stage verification pipeline + a structured problem→solution matrix.

**Conversion logic:** Rational credibility. Show the system's rigor, then invite users to test it ("test the verification system"). Appeals to analytical decision-makers.

**Emotional tone:** Structured, data-rich, system-authoritative. "Here's how it works — every step."

---

## Concept 3: High-Conviction Premium Trust Brand (`ramp/`)

**Inspiration:** Ramp

**Hero framing:** Bold imperative — "Stop guessing. Start trusting." Dark, oversized type, commanding.

**Section order:** Hero → Direct comparison table (us vs. everyone else) → Authority signals (by the numbers) → CTA

**Proof structure:** Competitive comparison table with 5 feature rows (source links, freshness, matching, reasoning, stale flagging) showing Hostliday ✓ vs Others ✗. Backed by 3 authority numbers.

**Conversion logic:** Binary choice + authority. Create a clear "us vs. them" frame, show definitive superiority, then present a decisive CTA ("The demo is live. See for yourself."). Urgency and conviction.

**Emotional tone:** Bold, decisive, premium. "We're right. Here's proof."

---

## Structural Differences Summary

| Dimension | Concept 1 (Intercom) | Concept 2 (Rippling) | Concept 3 (Ramp) |
|---|---|---|---|
| **Hero framing** | Personal introduction | Data dashboard | Bold imperative |
| **Proof model** | Live chat demo | Metrics + pipeline + table | Competitive comparison |
| **Section count** | 5 sections | 5 sections | 4 sections |
| **Section order** | Chat → Comparison → Promises | Stats → Pipeline → Problems | Comparison → Authority |
| **Conversion hook** | Emotional / experiential | Rational / systematic | Authoritative / decisive |
| **Tone** | Warm, conversational | Structured, analytical | Bold, commanding |
| **Dark/light** | Light (warm paper) | Light (clean surface) | Dark (premium) |

## Shared Constraints (All Concepts)

- **Single CTA:** "Try the demo" — no secondary CTAs
- **Brand identity:** Hostliday brand mark (H), Montserrat font, red palette (#be0000, #da0000, #810000)
- **Tagline preserved:** "Experiences, adventures, and stays"
- **Demo link:** All CTAs link to `../index.html#demo` (the working demo)
