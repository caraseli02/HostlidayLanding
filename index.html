Problem analysis

The product fixes trust failures, not just planning friction.

Search results force you to verify everything manually

Ratings and reviews create noise, but they do not explain why a hotel or activity is current, real, or right for this trip.

Recommendations expose their sources

Hostliday makes verification inspectable with direct source links and freshness evidence on every pick.

Source visibility

Generic AI outputs hide uncertainty

Confidence is graded, not faked

Freshness logic

Confidence model

The interface gets more explicit as certainty decreases.

Verified today

Confirmed within the last 24 hours, with matching source data and no unresolved conflicts.

Verified within 7 days

Recent enough to trust for planning, while still signaling that drift is possible.

Stale

Still shown if useful, but visibly marked so the user knows to double-check before relying on it.

Unverified

No live confirmation available. The system surfaces that uncertainty instead of hiding it behind polished copy.
